💻 КОД И СКРИПТЫ:

server.py              Основной серверный скрипт (446 строк)
                       ├─ Многопоточность
                       ├─ Логирование
                       ├─ Передача файлов
                       └─ Персистентное хранилище

client.py              Клиент для работы с файлами
                       ├─ Загрузка файлов

                       ├─ Создает пользователя
                       ├─ Копирует файлы

                       └─ Запускает сервер

manage.sh              Управление сервисом
                       ├─ start/stop/restart
                       ├─ status
                       ├─ logs
                       └─ enable/disable

logs.sh                Специализированный просмотр логов
                       ├─ follow (следить)
                       ├─ errors (ошибки)
                       ├─ users (активность)
                       ├─ files (передачи)
                       ├─ today (сегодня)
                       └─ stats (статистика)

serv_mess.service      Systemd конфиг (автозапуск)
                       └─ Копируется в /etc/systemd/system/

⚡ ЕЖЕДНЕВНЫЕ КОМАНДЫ:
═══════════════════════════════════════════════════════════════

Проверка работы:
  ./manage.sh status           Быстрая проверка
  ./manage.sh info             Полная информация
  ./logs.sh stats              Статистика

Просмотр логов:
  ./manage.sh logs             Последние 50 строк
  ./manage.sh logs -f          Следить в реальном времени
  ./logs.sh follow             То же самое
  ./logs.sh errors             Только ошибки
  ./logs.sh users              Активность пользователей
  ./logs.sh files              Передачи файлов
  ./logs.sh today              Сегодняшние логи

Управление:
  ./manage.sh restart          Перезагрузить
  ./manage.sh stop             Остановить
  ./manage.sh start            Запустить
  ./manage.sh enable           Включить автозапуск
  ./manage.sh disable          Отключить автозапуск

Работа с файлами:
  python3 client.py upload server:7002 file.txt
  python3 client.py download server:7002 file.txt

🆘 РЕШЕНИЕ ПРОБЛЕМ:
═══════════════════════════════════════════════════════════════

Сервис не запускается:
  $ sudo systemctl status serv_mess -l
  $ sudo journalctl -u serv_mess -n 50
  
  Полное решение: см. PRODUCTION.md → Решение проблем

Порт занят:
  $ sudo lsof -i :7002
  $ sudo kill -9 <PID>
  $ ./manage.sh restart

Нет прав доступа:
  $ sudo chown -R memo:memo /opt/serv_mess
  $ sudo chmod -R 755 /opt/serv_mess

Размер логов слишком большой:
  $ du -h /opt/serv_mess/logs/server.log
  $ sudo sh -c 'echo "" > /opt/serv_mess/logs/server.log'
  
  Или архивировать перед очисткой
